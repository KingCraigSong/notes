# 莫坊笔记整理规范

> 本规范定义了笔记的分类体系与写作标准

## 笔记分类

### 算法模型
**内容范围：** 算法原理、机器学习模型、深度学习模型等理论学习笔记

**二级分类建议：**
- `01_基础/` — 数据结构、基本算法
- `02_高级/` — 动态规划、图算法等
- `03_机器学习/` — 传统ML模型、特征工程
- `04_深度学习/` — CNN、RNN、Transformer等

**写作规范：**
- 概念定义 + 数学公式（如有）
- 时间/空间复杂度分析
- 伪代码或代码实现
- 适用场景与局限性

---

### 工程实践
**内容范围：** 编程技能、软件安装、运维部署、配置调试等工程实践笔记

**二级分类建议：**
- `01_编程语言/` — 按语言分类（Python/Java/Go/JS...）
- `02_框架工具/` — 主流框架、常用工具
- `03_软件安装/` — 数据库、中间件、开发工具
- `04_运维部署/` — Docker、K8s、CI/CD、监控
- `05_配置/` — 环境配置、项目配置

**写作规范：**
- 环境要求
- 完整安装/配置步骤
- 常见问题与解决方案
- 最佳实践总结

---

### 读书笔记
**内容范围：** 书籍阅读笔记，包括技术书籍和文学书籍

**二级分类建议：**
- `技术书籍/` — 按书籍名称或主题
- `文学书籍/` — 按书籍名称或主题

**写作规范：**
- 书籍基本信息（书名、作者、阅读时间）
- 章节概要与重点摘录
- 个人理解与思考
- 读后感或实践应用

---

### 模版代码
**内容范围：** 可直接复用的代码片段、配置模板

**二级分类建议：**
- `代码片段/` — 按语言/框架分类
  - `Python/`
  - `Java/`
  - `Go/`
  - `JavaScript/`
- `运维配置/` — Docker、K8s、Nginx等
- `项目模板/` — 项目初始化模板

**写作规范：**
- 适用场景说明
- 完整可运行的代码
- 关键注释说明
- 依赖环境标注

---

### 临时
**内容范围：** 临时想法、待整理笔记

**二级分类建议：**
- `待整理/` — 需要进一步整理的笔记
- `想法/` — 随手记下的想法

**写作规范：**
- 临时笔记用 `[[临时/待整理/]]` 标签标记
- 定期回顾整理，不长期堆积
- 整理完成后移到对应分类

---

## 通用写作规范

1. **文件名** — 使用英文或拼音，首字母小写，如 `二分查找.md`
2. **标签** — 使用 `tags` 标记分类，如 `tags: [算法, 排序]`
3. **引用** — 使用 `> ` 块引用标注重要内容
4. **代码块** — 必须标注语言类型
5. **图片** — 统一存放在同名附件目录
