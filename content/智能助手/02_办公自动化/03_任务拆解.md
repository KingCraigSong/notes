# 任务拆解

## 为什么需要任务拆解？

面对一个复杂的任务，直接让智能助手完成往往效果不佳。原因如下：

1. **信息过载**：一次性提供太多信息，智能助手可能无法准确理解全部需求
2. **缺乏细节**：复杂任务需要大量细节，一次性描述容易遗漏
3. **难以验证**：大任务的输出难以检查和验证
4. **耗时较长**：大任务响应时间长，容易出错

任务拆解就像是把一座大山拆成一个个小土坡，逐个攻克，最终完成整个任务。

## 任务拆解原则

### 原则 1：单一职责

每个子任务应该只有一个明确的目标：

```
❌ 不好：处理销售数据并生成报表

✅ 好：
   1. 清洗销售数据
   2. 计算销售汇总
   3. 生成 Excel 报表
```

### 原则 2：可验证性

每个子任务的结果应该是可验证的：

```
❌ 不好：优化代码

✅ 好：将代码执行时间从 10 秒优化到 3 秒以内
```

### 原则 3：循序渐进

按照逻辑顺序安排子任务，后一个任务依赖前一个任务的结果：

```
1. 读取原始数据
2. 数据清洗
3. 数据转换
4. 数据分析
5. 生成报告
```

### 原则 4：粒度适中

任务不能太大也不能太小：

```
❌ 太大：开发一个完整的管理系统
❌ 太小：定义一个变量

✅ 适中：实现用户登录功能
```

## 任务拆解方法

### 方法 1：功能拆解法

根据功能模块进行拆解：

```
任务：开发一个员工信息管理系统

拆解：
1. 数据库设计与创建
2. 员工信息录入功能
3. 员工信息查询功能
4. 员工信息修改功能
5. 员工信息删除功能
6. 数据导出功能
7. 测试与调试
```

### 方法 2：流程拆解法

按照业务流程进行拆解：

```
任务：处理月度销售报表

拆解：
1. 收集各部门销售数据
2. 合并所有数据到一个文件
3. 数据清洗（删除空值、格式统一）
4. 计算汇总数据
5. 生成可视化图表
6. 撰写分析报告
7. 发送给相关人员
```

### 方法 3：问题拆解法

针对问题进行拆解：

```
任务：优化 Excel 处理脚本的性能

拆解：
1. 分析当前脚本的执行时间
2. 找出性能瓶颈
3. 优化数据读取方式
4. 优化数据处理逻辑
5. 优化内存使用
6. 测试优化后的性能
```

## 任务分配实战

### 实战场景 1：批量处理 Excel 文件

**原始任务：**
```
我有 100 个 Excel 文件，每个文件有销售数据。请帮我合并这些文件，并生成月度销售报告。
```

**拆解后的任务分配：**

**任务 1：了解文件结构**
```
我有 100 个 Excel 文件在文件夹"销售数据"中。请帮我写一个 Python 脚本，列出所有文件的名称、大小和列名，让我了解数据结构。
```

**任务 2：合并文件**
```
好的，所有文件都有相同的列结构：日期、地区、产品、金额、数量。请帮我写一个脚本，将这 100 个文件合并成一个 Excel 文件。
```

**任务 3：数据清洗**
```
合并完成后，数据中有一些问题：
1. 日期格式不统一（有的是"2024/1/1"，有的是"2024-01-01"）
2. 金额列有非数值数据
3. 存在重复行

请帮我写一个脚本进行数据清洗。
```

**任务 4：生成报告**
```
数据清洗完成，请帮我生成月度销售报告，包含：
1. 各地区销售汇总
2. 各产品销售排名
3. 每日销售趋势图表
4. 销售同比增长分析
```

### 实战场景 2：自动化周报生成

**原始任务：**
```
请帮我实现周报自动化，每天自动收集数据，每周生成一份周报。
```

**拆解后的任务分配：**

**任务 1：需求分析**
```
我想实现周报自动化。请帮我分析需要哪些步骤，以及每个步骤的具体内容。

周报内容包括：
- 本周工作完成情况
- 下周工作计划
- 遇到的问题和解决方案
- 项目进度跟踪

数据来源：
- 我的待办事项清单
- 项目管理工具（如 Jira）
- 邮件记录
```

**任务 2：数据收集**
```
请帮我写一个 Python 脚本，用于收集周报所需的数据：
1. 从待办事项清单中读取已完成的任务
2. 从 Jira API 获取项目进度
3. 统计本周发送和接收的邮件数量

输出格式：JSON 文件
```

**任务 3：报告生成**
```
请帮我写一个脚本，根据收集到的数据生成周报文档：
1. 使用 Python-docx 生成 Word 文档
2. 包含标题、正文、表格和图表
3. 按照固定模板格式化
```

**任务 4：自动化调度**
```
请帮我设置一个定时任务，每周五下午 5 点自动运行周报生成脚本，并将生成的周报发送到我的邮箱。
```

## 任务分配技巧

### 技巧 1：明确输入输出

每次分配任务时，明确告诉智能助手输入是什么，期望的输出是什么：

```
输入：100 个 Excel 文件，每个文件包含销售数据
输出：一个合并后的 Excel 文件，包含所有数据
```

### 技巧 2：提供约束条件

告诉智能助手任务的约束条件：

```
约束条件：
1. 每个文件大小不超过 10MB
2. 处理时间不能超过 5 分钟
3. 不能修改原始文件
```

### 技巧 3：设定优先级

当有多个任务时，设定优先级：

```
任务优先级：
1. 高优先级：数据清洗脚本（明天就要用）
2. 中优先级：报告生成脚本（本周内完成）
3. 低优先级：自动化调度（下周完成）
```

### 技巧 4：逐步验证

每个子任务完成后，先验证结果，再进行下一个任务：

```
任务 1 完成 → 验证数据是否正确 → 任务 2 → 验证 → 任务 3 → ...
```

### 技巧 5：及时反馈

如果智能助手的输出不符合预期，及时反馈：

```
这个脚本运行后，合并的数据有重复。请帮我检查一下代码，看看哪里出了问题。
```
