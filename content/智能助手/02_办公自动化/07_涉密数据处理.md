# 涉密数据处理

## 直接使用 AI 智能体处理涉密信息的危害性

在办公自动化过程中，我们经常需要处理各种敏感数据，如员工工资、客户信息、财务数据等。直接将这些涉密数据提供给 AI 智能体处理，存在以下风险：

### 风险 1：数据泄露

AI 智能体通常需要联网运行，你的数据会上传到服务器进行处理。这意味着：
- 敏感数据可能被第三方获取
- 数据可能被用于模型训练
- 数据可能被存储在云端，存在被盗取的风险

### 风险 2：合规问题

许多行业（如金融、医疗、政府）都有严格的数据保护法规：
- 《个人信息保护法》
- 《网络安全法》
- GDPR（欧盟通用数据保护条例）

违规处理涉密数据可能面临罚款、法律责任等严重后果。

### 风险 3：不可控的输出

AI 智能体的输出是基于训练数据生成的，可能存在：
- 数据被意外泄露到输出中
- 敏感信息被写入日志或缓存
- 输出内容被共享或传播

### 风险 4：缺乏审计追踪

直接使用 AI 处理数据时，你无法完全控制：
- 数据的流向
- 数据的使用方式
- 是否有备份或副本

## 如何正确使用 AI 智能体处理敏感数据

### 核心原则：数据不上传，脚本本地跑

这是处理敏感数据的基本原则：

```
安全流程：
1. 准备模板和测试数据
2. 向 AI 描述需求（只提供模板和测试数据）
3. 获取生成的脚本
4. 本地验证脚本
5. 使用真实数据本地运行
```

### 方法 1：使用模板和测试数据

**步骤：**

1. **准备模板文件**：创建不含真实数据的模板，使用占位符表示数据位置

```
┌─────────────────────────────────────────┐
│              工资条                     │
├──────────┬──────────────────────────────┤
│ 项目     │ 金额                         │
├──────────┼──────────────────────────────┤
│ 姓名     │ {{姓名}}                     │
│ 工号     │ {{工号}}                     │
│ 部门     │ {{部门}}                     │
│ 基本工资 │ {{基本工资}}                  │
└──────────┴──────────────────────────────┘
```

2. **创建测试数据**：使用虚构数据创建测试文件

| 姓名 | 工号 | 部门 | 基本工资 |
| :--- | :--- | :--- | :--- |
| 张三 | 001 | 技术部 | 10000 |
| 李四 | 002 | 市场部 | 8000 |

3. **描述需求**：向 AI 描述需求时，只提供模板和测试数据
4. **获取脚本**：AI 生成处理脚本
5. **本地验证**：使用测试数据验证脚本正确性
6. **本地运行**：切换到真实数据，在本地运行脚本

### 方法 3：数据脱敏处理

在开发和测试阶段，对数据进行脱敏处理：

```python
def mask_sensitive_data(data):
    masked_data = data.copy()
    
    if '姓名' in masked_data.columns:
        masked_data['姓名'] = masked_data['姓名'].apply(
            lambda x: x[0] + '*' * (len(x) - 1) if isinstance(x, str) else x
        )
    
    if '工号' in masked_data.columns:
        masked_data['工号'] = masked_data['工号'].apply(
            lambda x: '*' * (len(str(x)) - 2) + str(x)[-2:] if x else x
        )
    
    if '手机号' in masked_data.columns:
        masked_data['手机号'] = masked_data['手机号'].apply(
            lambda x: str(x)[:3] + '****' + str(x)[-4:] if x else x
        )
    
    return masked_data
```

### 方法 4：日志脱敏

确保日志中不记录敏感信息：

```python
import logging

def log_with_mask(message, sensitive_data=None):
    if sensitive_data:
        for key, value in sensitive_data.items():
            message = message.replace(str(value), '***')
    logging.info(message)

log_with_mask(f"处理用户: {user_name}", {'user_name': user_name})
```

确保脚本可以在离线环境下运行：

### 方法 6：清理临时文件

脚本运行完成后，清理临时文件：

```python

        temp_file.write(b"data")

def cleanup_temp_files(folder_path):
    for filename in os.listdir(folder_path):
        if filename.startswith('temp_') or filename.endswith('.tmp'):
            file_path = os.path.join(folder_path, filename)
            os.remove(file_path)
```

## 数据安全检查清单

在处理涉密数据前，请检查以下事项：

### ✅ 数据检查

- [ ] 确认数据是否包含敏感信息（姓名、工号、手机号、身份证号等）
- [ ] 确认数据是否需要脱敏处理
- [ ] 确认数据是否已备份

### ✅ 环境检查

- [ ] 确认脚本在本地运行
- [ ] 确认网络已断开（处理涉密数据时）
- [ ] 确认没有上传数据的操作

### ✅ 脚本检查

- [ ] 确认脚本没有发送数据到外部服务器
- [ ] 确认脚本没有记录敏感信息到日志
- [ ] 确认脚本使用配置文件管理路径

### ✅ 输出检查

- [ ] 确认输出文件不包含额外的敏感信息
- [ ] 确认输出文件保存在安全位置
- [ ] 确认临时文件已清理

## 实战示例：安全处理工资数据

### 场景

HR 部门需要每月处理员工考勤数据并生成工资条。数据包含员工姓名、工号、部门、基本工资、绩效奖金等敏感信息。

### 安全处理流程

**步骤 1：准备模板**

创建工资条模板 `工资模板.xlsx`，使用占位符。

**步骤 2：创建测试数据**

创建 `测试数据.xlsx`，包含虚构数据。

**步骤 3：向 AI 描述需求**

```
我需要你帮我写一个 Python 脚本，用于批量生成员工工资条。

【输入文件】
1. 考勤数据.xlsx - 包含员工考勤信息，列如下：
   - 姓名、工号、部门、基本工资、绩效奖金、迟到次数、早退次数、请假天数

2. 工资模板.xlsx - 工资条模板，使用 {{字段名}} 作为占位符

【计算规则】
1. 考勤扣款 = 迟到次数 × 50 + 早退次数 × 50 + 请假天数 × 200
2. 应发工资 = 基本工资 + 绩效奖金
3. 实发工资 = 应发工资 - 考勤扣款

【输出要求】
1. 为每个员工生成一个独立的 Excel 文件
2. 文件名格式：工资条_姓名_工号.xlsx
3. 将文件保存到"工资条"文件夹中

【技术要求】
1. 使用 pandas 和 openpyxl 库
2. 使用配置字典管理文件路径
3. 处理过程中打印进度信息
```

**步骤 4：获取脚本**

AI 生成脚本，包含配置字典：

```python
CONFIG = {
    'input_file': '测试数据.xlsx',
    'template_file': '工资模板.xlsx',
    'output_folder': '工资条'
}
```

**步骤 5：本地验证**

使用测试数据运行脚本，验证输出是否正确。

**步骤 6：切换真实数据**

修改配置：

```python
CONFIG = {
    'input_file': '真实数据.xlsx',
    'template_file': '工资模板.xlsx',
    'output_folder': '工资条'
}
```

**步骤 7：离线运行**

断开网络，运行脚本处理真实数据。

## 总结

处理涉密数据的核心原则是：**数据不上传，脚本本地跑**。通过使用模板和测试数据、配置文件管理、数据脱敏、离线运行等方法，可以在享受 AI 带来的效率提升的同时，确保数据安全。
