# 通用企业级工作流平台业务需求全景图

以下是一份"通用企业级工作流平台"的业务需求全景图，完全从"业务用户"视角梳理，可直接拿这张图去拉业务方、HR、财务、法务、运维一起评审。

## 一、流程建模阶段（设计器）

| 业务需求 | 用户故事 | 验收要点 |
| :--- | :--- | :--- |
| **1. 拖拽式流程图** | 作为流程管理员，我希望像 Visio 一样拖拽节点，10 分钟完成"采购申请"流程图。 | 支持开始、结束、用户任务、服务任务、网关节点；支持子流程、调用活动；支持 BPMN 2.0 规范导出。 |
| **2. 节点属性可视化配置** | 我希望在画板上直接设置"审批人、表单、时限、通知、按钮文案"，无需 XML。 | 右侧属性面板 ≥20 项：候选人规则、会签策略、驳回范围、自动通过策略、到期提醒、字段权限。 |
| **3. 版本管理** | 流程上线后，我希望发布 V2.0 时老实例继续按 V1.0 跑，新实例按 V2.0 跑。 | 支持"发布/草稿"双版本；支持一键对比两个版本差异（节点、连线、表单）。 |
| **4. 模拟试运行** | 流程发布前，我希望输入任意角色/表单数据，系统能自动跑一遍并标红卡住的位置。 | 提供"沙箱运行"按钮，5 秒内返回模拟结果；支持断点单步调试。 |

## 二、组织与人（Who）

| 业务需求 | 用户故事 | 验收要点 |
| :--- | :--- | :--- |
| **5. 多维度候选人** | 审批人可能是"直属领导"、"角色"、"岗位"、"表单变量"、"SQL 规则"、"外部接口"。 | 每种规则都能在前端下拉框选到；支持组合规则"且/或"。 |
| **6. 代理/转办/协办** | 当我请假时，可把我的审批权代理给 B；审批途中可转给 C；需要财务协办时加签 D。 | 代理支持时间段；转办记录可查；协办支持串并行。 |
| **7. 角色矩阵** | 公司 2000 人，矩阵式管理，我希望按"部门+项目"二维角色自动找到审批人。 | 支持 Excel 批量导入角色矩阵；流程节点可按矩阵行列定位。 |

## 三、表单与数据（What）

| 业务需求 | 用户故事 | 验收要点 |
| :--- | :--- | :--- |
| **8. 低代码表单设计器** | 作为 HR，我希望拖 10 个控件就能做出"请假单"，并与流程绑定。 | 控件 ≥20 种：单行文本、金额、附件、明细表、电子签章；支持手机端自适应。 |
| **9. 字段权限** | 在"报销单"里，财务节点才能看到"发票号码"，员工不能看。 | 按节点设置可见/可编辑/必填；权限冲突时给出提示。 |
| **10. 外部数据联动** | 选择"供应商编号"后，系统自动带出银行账号、税率。 | 支持 REST、数据库、WebService 三种数据源；可配置刷新频率。 |
| **11. 打印/水印/归档** | 审批完成后，系统能生成带二维码的 PDF，自动加盖电子章并推送到档案系统。 | PDF 模板可 Word 上传；支持骑缝章；归档接口支持 FTP、SharePoint、Kingdee。 |

## 四、流程运行（How）

| 业务需求 | 用户故事 | 验收要点 |
| :--- | :--- | :--- |
| **12. 提交 & 撤回** | 员工提交后，下一节点未处理前可撤回修改。 | 撤回后流程实例回退到草稿，通知已发待办自动消失。 |
| **13. 同意/驳回/加签** | 审批人可一键同意，也可驳回到任意节点，或加签其他人。 | 驳回支持"原路返回"和"指定节点"；加签支持串行、并行、或签。 |
| **14. 自动通过** | 当金额 <1000 元且部门=行政部时，系统自动同意。 | 配置 Groovy/MVEL 表达式；支持调用外部风控接口。 |
| **15. 会签/或签** | 总经理节点需要 5 位副总"全部同意"才通过；或任意 1 人同意即可。 | 支持串行、并行、比例（≥60%）、绝对数量。 |
| **16. 超时提醒 & 升级** | 节点超过 48 小时未处理，系统先提醒，再 24 小时后自动升级给中心负责人。 | 提醒渠道：邮件、企业微信、短信；升级规则可配。 |
| **17. 评论 & 附件** | 审批人可填写评论并上传附件，后续节点可见。 | 评论支持 @人；附件支持在线预览 Office、PDF。 |

## 五、监控与分析

| 业务需求 | 用户故事 | 验收要点 |
| :--- | :--- | :--- |
| **18. 个人工作台** | 作为普通员工，我登录后能看到"我的待办、已办、我发起的、抄送给我"。 | 4 个 Tab；支持关键词搜索；支持批量审批（同类型≤20 条）。 |
| **19. 流程跟踪图** | 我想看到绿色节点表示已通过，红色表示当前停留，灰色表示未走到。 | 实时刷新；支持手机端手势缩放；支持点击节点查看审批记录。 |
| **20. 超时排名** | 作为行政主管，我想知道哪类流程平均耗时最长，哪个员工经常超时。 | 报表支持按部门、流程、人三维钻取；可导出 Excel。 |
| **21. 瓶颈分析** | 系统能自动算出"等待时间/处理时间"比例最高的 3 个节点，并给出优化建议。 | 采用桑基图展示；可下钻到实例。 |

## 六、权限与合规

| 业务需求 | 用户故事 | 验收要点 |
| :--- | :--- | :--- |
| **22. 分级管理员** | 总部管理员能管所有流程，分公司管理员只能管分公司范围内的流程。 | 支持数据权限行级过滤；支持功能按钮级权限。 |
| **23. 审计日志** | 任何"审批、驳回、转办、修改流程"都要留痕，满足 SOX/ISO27001。 | 日志不可篡改，写入 ES 并每日 Hash 上链；保存 ≥10 年。 |
| **24. 电子签名 & 数字证书** | 关键节点（如法人盖章）必须使用 UKey 签名。 | 支持 RSA/SM2；支持国密证书；签名数据嵌入 PDF。 |

## 七、集成与扩展

| 业务需求 | 用户故事 | 验收要点 |
| :--- | :--- | :--- |
| **25. 单点登录** | 与公司 AD、钉钉、飞书、企业微信集成，登录后免跳转。 | 支持 OAuth2、CAS、SAML2；手机端自动获取免登 code。 |
| **26. 消息集成** | 待办消息能推送到钉钉群、飞书群、企业微信待办。 | 支持 Markdown、@人；消息可点击直达详情页。 |
| **27. 业务系统回调** | 审批完成后，系统自动把状态写回 ERP/CRM/SRM。 | 提供 WebHook、Kafka、RocketMQ 三种方式；失败重试 ≥3 次。 |
| **28. 低代码插件** | 如果系统内置功能不够，我可以用 JavaScript 写脚本扩展。 | 支持 Node.js 沙箱；支持上传 npm 包；脚本错误不影响主流程。 |

## 八、运维与运营

| 业务需求 | 用户故事 | 验收要点 |
| :--- | :--- | :--- |
| **29. 多租户** | 集团下属 5 家公司共用 1 套系统，数据互不可见。 | 按租户分库/分表/分 Schema；支持租户级别白标域名、LOGO。 |
| **30. 灰度发布** | 新版本流程先给 10% 员工试用，无问题再全量。 | 支持按部门、角色、人三维流量分配；可一键回滚。 |
| **31. 性能水位** | 系统需要支持 2 万员工，日常 1 万流程实例/天，峰值 3 倍。 | 压测报告：CPU≤60%，RT≤500ms，99 线≤1s；历史数据 10 年可归档。 |
| **32. 灾备** | 机房级故障 30 分钟内可切换，数据零丢失。 | 采用 MySQL Group Replication + Keepalived；做双活写入冲突检测。 |

## 使用方法

1. 打印这张表，拉业务、HR、财务、法务、IT、运维一起开 2 小时评审会。
2. 每行打钩/叉，标优先级 P0、P1、P2。
3. 把 P0 功能剪出来，就是 MVP（最小可用产品）范围，通常 6~8 周可上线。
4. 剩余功能按迭代排期，每 2 周一个版本。
