对话技巧

为什么对话技巧很重要？

与智能助手的沟通质量直接影响到它能否准确理解你的需求。掌握好的对话技巧，可以让你：

  • 更快地得到想要的结果
  • 减少不必要的重复沟通
  • 提高脚本生成的准确性
  • 更好地利用智能助手的能力

有效提问技巧

技巧 1：明确需求

❌ 不好的提问：

帮我处理一下这个表格。

✅ 好的提问：

我有一个 Excel 文件"销售数据.xlsx"，里面有 1000 行数据。请帮我：
1. 删除所有空行
2. 将"金额"列格式化为数值
3. 筛选出"地区"为"华东"的记录
4. 计算每个产品的总销售额
5. 生成一个数据透视表

技巧 2：提供上下文

❌ 不好的提问：

这个代码有问题。

✅ 好的提问：

我写了一个 Python 脚本，想读取 Excel 文件并计算平均值，但运行时出现了错误。

代码如下：
import pandas as pd
df = pd.read_excel("data.xlsx")
average = df["金额"].mean()
print(average)

错误信息：
AttributeError: 'DataFrame' object has no attribute 'mean'

请帮我看看哪里出错了，如何修复？

技巧 3：分步提问

对于复杂任务，不要一次性问完所有问题，可以分步进行：

第一步：

我想批量重命名文件夹中的所有文件，按照"日期_序号_名称"的格式。请告诉我需要怎么做。

第二步：

好的，我明白了流程。现在请帮我写一个 Python 脚本来实现这个功能。

第三步：

脚本写好了，但是运行时报错了。错误信息是 ... 请帮我排查一下问题。

技巧 4：使用结构化格式

使用列表、表格等格式让你的需求更清晰：

请帮我设计一个员工信息管理系统，包含以下功能：

【功能需求】
1. 员工信息录入（姓名、工号、部门、职位、入职日期、薪资）
2. 员工信息查询（按姓名、部门、工号搜索）
3. 员工信息修改
4. 员工信息删除
5. 数据导出（Excel 格式）

【技术要求】
- 使用 Python 编写
- 使用 SQLite 数据库
- 命令行界面即可

【输出要求】
- 完整的源代码
- 使用说明

提示词模板

模板 1：写代码

请帮我写一个 [语言] 脚本，用于 [功能描述]。

要求：
1. [具体要求 1]
2. [具体要求 2]
3. [具体要求 3]

输入：[输入数据描述]
输出：[输出结果描述]

模板 2：解释概念

请用通俗易懂的语言解释什么是 [概念名称]。

要求：
1. 用生活中的例子来说明
2. 不要使用太多专业术语
3. 分点说明

模板 3：排查问题

我遇到了一个问题，请帮我分析并解决。

问题描述：
[详细描述问题现象]

相关代码/日志：
[粘贴代码或错误日志]

我已经尝试过：
1. [尝试方案 1]
2. [尝试方案 2]

请帮我找出问题原因并提供解决方案。

模板 4：优化建议

我有一个 [项目/流程/代码]，请帮我分析并提供优化建议。

当前情况：
[描述当前状态]

目标：
[希望达到的目标]

约束条件：
[时间、资源等限制]

请提供具体的优化建议。

进阶提示词技巧

技巧 1：使用角色设定

给智能助手设定一个特定的角色，可以让它更好地理解你的需求：

你现在是一名资深的 Python 数据分析专家，请帮我优化这个数据处理脚本。

脚本如下：
[粘贴你的代码]

请从以下几个方面进行优化：
1. 性能优化（减少执行时间）
2. 代码可读性（添加注释、优化结构）
3. 错误处理（添加异常捕获）
4. 内存优化（处理大文件时的内存使用）

技巧 2：提供示例输出

给出期望的输出示例，可以让智能助手更准确地理解你的需求：

请帮我写一个 Python 脚本，将 JSON 数据转换为 CSV 格式。

输入数据格式：
[
  {"name": "张三", "age": 25, "city": "北京"},
  {"name": "李四", "age": 30, "city": "上海"}
]

期望输出格式（CSV）：
name,age,city
张三,25,北京
李四,30,上海

请写完整的脚本代码。

技巧 3：使用约束条件

明确告诉智能助手哪些是不能做的：

请帮我写一个 Python 脚本，爬取网站数据。

【要求】
1. 使用 requests 库
2. 爬取间隔至少 1 秒（避免被封禁）
3. 不要爬取超过 100 页的数据
4. 遇到错误时自动重试，最多重试 3 次
5. 将数据保存为 JSON 格式

【禁止】
1. 不要使用 Scrapy 框架
2. 不要并发请求（避免服务器压力）
3. 不要爬取付费内容

技巧 4：分阶段提示

对于复杂任务，可以分阶段给出提示：

阶段 1：分析需求
我想实现一个自动化测试工具，请帮我分析需要哪些功能模块。

阶段 2：设计架构
根据分析结果，请帮我设计系统架构，包含模块划分和数据流。

阶段 3：编写代码
请帮我实现核心模块的代码。

阶段 4：测试优化
请帮我编写测试用例，并优化代码。

注意事项

  1. 保持礼貌：友好的交流有助于获得更好的结果
  2. 耐心等待：复杂任务可能需要较长时间处理
  3. 及时反馈：如果回答不符合预期，请及时告诉它
  4. 保护隐私：不要发送敏感信息，如密码、身份证号等
  5. 备份数据：运行脚本前，建议备份原始数据
